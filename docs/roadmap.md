# InvestorFrame Roadmap

This roadmap outlines the expected evolution of InvestorFrame from an early public scaffold into a more complete event-driven market intelligence product.

The roadmap is intentionally structured around product coherence and trust, not feature inflation.

---

## Current Stage

InvestorFrame is in an early public stage.

What already exists in some form:

- project identity
- product framing
- repo skeleton
- initial README
- dashboard and scenario concepts
- design system direction
- landing page direction
- event-driven workflow definition

What still needs polishing:

- repo hygiene
- docs completeness
- examples
- visual assets
- product consistency
- public presentation maturity

---

## Phase 1 — Public Repo Hardening

### Goal
Make the repository feel like a serious, maintainable, public open-source project.

### Focus
- refine About metadata
- improve README first impression
- add docs
- add examples
- add contributing and security guidance
- normalize branch and release hygiene
- improve public trust signals

### Deliverables
- `CONTRIBUTING.md`
- `SECURITY.md`
- `CHANGELOG.md`
- architecture docs
- roadmap docs
- example outputs
- better screenshots / visual assets

---

## Phase 2 — Product Surface Clarity

### Goal
Make InvestorFrame understandable in one pass.

### Focus
- tighten product language
- clarify the three main surfaces:
  - Dashboard
  - Scenario Lab
  - Reports & Explain
- make the workflow highly legible
- align repo, docs, and site around one narrative

### Deliverables
- polished landing page
- product mockups
- workflow diagram
- consistent UI language
- consistent brand system

---

## What InvestorFrame Should Not Become

To preserve clarity, InvestorFrame should resist drifting into:

- generic stock screener territory
- broker UI imitation
- certainty-marketing finance product
- hype-driven "AI alpha" branding
- overcomplicated feature sprawl
- unexplainable black-box outputs

The project should remain centered on:

- framing the market
- understanding transmission
- testing scenarios
- improving interpretation

---

## Guiding Principle

The goal is not to build the loudest finance product.

The goal is to build a calmer, clearer, more structured way to understand how events may reshape the market.

That is the core promise of InvestorFrame.
